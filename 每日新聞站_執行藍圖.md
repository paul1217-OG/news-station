## 一、誠實建議：你的選項組合需要調整

你選了「不懂技術」+「VPS 雲端伺服器」這兩個選項實際上會打架。VPS 對非技術背景者門檻偏高（要會 SSH、Linux、Docker、防火牆、cronjob、憑證更新），故障時又沒有客服。

我列三條路線給你選，**強烈建議 A 路線**：

| 路線 | 適合對象 | 月成本 | 學習門檻 | 推薦度 |
|---|---|---|---|---|
| **A. n8n Cloud + 外掛 AI** | 不懂技術但想自己掌控 | NT$600～900 | 低（拖拉式介面） | ★★★★★ |
| B. Make.com（前 Integromat） | 完全不想碰任何技術 | NT$300～700 | 最低 | ★★★★☆ |
| C. VPS 自架（你原本選的） | 想學技術或長期省錢 | NT$200～600 | 高 | ★★☆☆☆ |

**為什麼推 A 路線（n8n Cloud）：**
- 拖拉式 workflow 介面，每個節點是一個動作（抓 RSS、呼叫 GPT、發 LINE），看得到流程
- 內建超過 400 個整合（含 LINE、OpenAI、RSS、HTTP）
- 出錯時介面會直接標紅該節點，不用看 log
- 之後想學技術，n8n 也支援 JavaScript code node，可以漸進升級
- 升級到自架版（Self-hosted）只要把 workflow 匯出，無 vendor lock-in

下面所有規劃以 **n8n Cloud** 為基礎，但所有資料來源、提示詞、訊息範本同樣適用於 Make.com 或其他平台。

## 四、LINE 官方帳號從零建立（你目前在這一步）

### 4.1 申請官方帳號（30 分鐘）
1. 前往 https://www.linebiz.com/tw/entry/
2. 點「免費試用」→ 用個人 LINE 帳號登入
3. 選擇帳號類型：**未認證帳號**（免費，先用這個）
4. 填寫：帳號名稱、行業類別（選「媒體與資訊」）、頭像
5. 完成後進入 LINE Official Account Manager

### 4.2 啟用 Messaging API（20 分鐘）
1. 在 OA Manager → 設定 → Messaging API → **啟用 Messaging API**
2. 系統會要求建立 Provider（提供者），自由命名（例：「我的新聞站」）
3. 跳轉到 **LINE Developers Console** (https://developers.line.biz)
4. 在你的 Channel 找到「Channel access token (long-lived)」→ 點 **Issue** 取得 token
5. **重要：** 把 token 複製存好（這是 n8n 推播用的密鑰）
6. 切到「Messaging API」分頁，把「Auto-reply messages」「Greeting messages」依需要關閉

### 4.3 免費方案限制
| 方案 | 月費 | 推播訊息數 |
|---|---|---|
| 輕用量（免費） | NT$0 | 200 則/月 |
| 低用量 | NT$200 | 4,000 則/月 |
| 中用量 | NT$800 | 25,000 則/月 |

> **算給你聽：** 一天 1 則推播 × 30 天 = 30 則人次，但實際是 30 則 × 訂閱者人數。10 個朋友訂閱就是 300 則，已超過免費額度。**初期建議：** 先用免費方案測試 30 天，穩定後升級到低用量方案（NT$200/月）。

## 六、NotebookLM 整合方式（實話實說）

NotebookLM 目前 **沒有公開 API**，無法直接被 n8n 自動呼叫。整合做法有三種：

| 做法 | 自動化程度 | 備註 |
|---|---|---|
| **A. 用 GPT-4 / Claude 取代** | 全自動 | 推薦：API 成熟，效果不輸 NotebookLM |
| B. NotebookLM 手動每週更新「來源庫」 | 半自動 | 你手動把當週重要文件放進 NotebookLM，每日只用 GPT 摘要當日新聞 |
| C. 用 Selenium 爬 NotebookLM 介面 | 全自動但脆弱 | 不推薦：違反 ToS、介面改版即壞 |

**我的建議：** 走 A 路線（用 Claude/GPT API 做摘要），把 NotebookLM 留給你自己**深度研究**用。系統推播是「即時資訊」，NotebookLM 強在「跨多文件深度問答」，定位不同。

## 八、關鍵 Prompt 範例（給 AI 摘要節點用）

```
你是一位專業財經新聞編輯。以下是過去 24 小時從多個來源抓取的新聞標題與摘要，請執行：

1. **過濾**：剔除與台股、美股、亞洲股市、產業科技、宏觀經濟「無關」的內容
2. **去重**：合併報導同一事件的多篇文章
3. **排序**：依「對台灣投資人重要性」由高到低排序
4. **摘要**：選出最重要的 6～8 條，每條輸出：
   - emoji + 一句標題（不超過 20 字）
   - 2 句重點摘要（為什麼重要、可能影響）
   - 原文連結

5. **分類標籤**：每條標註類別 [台股/美股/亞股/歐股/產業/宏觀]

輸出格式：JSON 陣列，每個物件包含 {emoji, title, summary, link, tag}

【新聞素材】
{{input}}
```

## 十一、常見問題

**Q1：可不可以用免費方案？**
n8n 有自架免費版，但對非技術者門檻太高。Make.com 有 1,000 次/月免費操作，可以試試但每日推播很容易超量。

**Q2：AI 會不會抓錯重點或亂寫？**
會。所以建議前 2 週用「半自動」模式，你每天早上 7:00 看一眼草稿、按確認才發送。穩定後再切全自動。

**Q3：圖文要長什麼樣？**
建議三種版本各做 5 張測試卡片：
- 純文字＋數據圖表（最快，一張 2 秒生成）
- Canva 套版（精緻，每張 3～5 秒）
- AI 生圖（獨特，每張 15～30 秒，成本高）

讓 5 個朋友投票哪個版本他們最想每天看，再決定。

**Q4：如果某天新聞很少怎麼辦？**
Prompt 裡加「若重要新聞不足 4 條，補入產業趨勢分析或本週經濟數據預告」備援邏輯。

**Q5：法律與版權？**
標題＋ AI 重寫摘要＋連結原文是合理使用。**不要做的事：** 全文轉貼、移除來源、把 AI 摘要包裝成自己原創獨家。
